Introduction

The TJPA Helmify Engine provides a set of REST endpoints that allow automated pipelines, custom scripts, and integration services to generate or preview Helm charts programmatically from Kubernetes/OpenShift resource manifests or structured configs.

Note: All API requests are fully offline-compatible. The engine generates chart outputs using in-memory templating models and does not communicate with external registries during runtime.

Base URL

When running the container locally or in OpenShift, endpoints are exposed relative to your host URL (e.g. http://localhost:8080).

Generate packaged Helm Chart

Translates Kubernetes resource manifests and downloads a packed, gzip-compressed tar archive (.tar.gz) containing the completed chart layout.

POST /v1/generate
Request Headers
Header Type Description
X-Chart-Name string Required. The name of the target chart (e.g. portal-certidao).
X-Crd boolean Whether to extract Custom Resource Definitions (default: false).
X-Cert-Manager-Subchart boolean Add Cert-Manager subchart settings (default: false).
X-Cert-Manager-Install-Crd boolean Install Cert-Manager CRDs automatically (default: false).
X-Add-Webhook-Option boolean Enable Webhook support validation logic (default: false).
X-Optional-Crds boolean Allow toggling CRD install dynamically via values (default: false).
X-Generate-All-Templates boolean Generate all standard boilerplate templates (CM, Secret, Routes) for all components (default: false).
X-Dev-Repo-Url string TJPA developer source repository URL annotation for Chart.yaml (default: "").
Request Body

Raw string of the source Kubernetes resource manifests (YAML format). Multiple resources should be delimited by --- lines.

Example Command (Local Manifest)
cURL 
curl -X POST "http://localhost:8080/v1/generate" \
  -H "X-Chart-Name: portal-certidao" \
  -H "X-Crd: false" \
  -H "Content-Type: text/yaml" \
  --data-binary @deployment.yaml \
  --output portal-certidao.tar.gz
Example Command (Kustomize Output Pipe)
cURL 
kustomize build config/default | curl -X POST "http://localhost:8080/v1/generate" \
  -H "X-Chart-Name: portal-certidao" \
  -H "X-Generate-All-Templates: true" \
  -H "X-Dev-Repo-Url: https://github.com/my-org/my-app" \
  -H "Content-Type: text/yaml" \
  --data-binary @- \
  --output portal-certidao.tar.gz

Download Chart Wizard

Processes a structured JSON configuration payload, translates it to Helm chart resources, and generates a packaged, gzip-compressed tarball (.tar.gz) stream for download.

POST /v1/generate-wizard
Request Body (JSON)
JSON Body structure 
{
  "chartName": "my-service",
  "type": "single",
  "globalConfig": {
    "TZ": "America/Belem"
  },
  "deployments": {
    "frontend": {
      "replicas": 2,
      "image": {
        "repository": "quay.enterprise.tjpa.jus.br/devops/nginx",
        "tag": "1.25"
      },
      "service": {
        "port": 80
      },
      "cm": {
        "DB_USER": "admin"
      },
      "secret": {},
      "route": {
        "path": "/",
        "default": {
          "enabled": true,
          "host": "my-service.apps.ocp.tjpa.jus.br"
        }
      }
    }
  }
}
Example Command
cURL 
curl -X POST "http://localhost:8080/v1/generate-wizard" \
  -H "Content-Type: application/json" \
  -d '{
    "chartName": "my-service",
    "type": "single",
    "deployments": {
      "frontend": {
        "replicas": 2,
        "image": {"repository": "nginx", "tag": "alpine"},
        "service": {"port": 80}
      }
    }
  }' \
  --output my-service.tar.gz
